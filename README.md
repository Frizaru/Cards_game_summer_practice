# Техническое задание: игра Cards на Pygame

## 1. Общая информация

**Название проекта:** Cards  
**Жанр:** Логическая игра на запоминание  
**Платформа:** Desktop  
**Язык программирования:** Python  
**Библиотека:** Pygame  

## 2. Цель проекта

Разработать простую игру Cards, в которой игрок должен находить пары одинаковых карточек на игровом поле.

Перед началом игры пользователь должен попадать в главное меню, где он сможет выбрать один из встроенных наборов изображений для карточек.

Примеры наборов:

- персонажи из сериала;
- мемы;
- животные;
- любые другие тематические изображения.

После выбора набора запускается игровое поле с карточками выбранной тематики.

## 3. Основная идея игры

На игровом поле расположены закрытые карточки. Игрок открывает две карточки за один ход.

Если изображения на карточках совпадают, карточки остаются открытыми.  
Если изображения не совпадают, карточки через короткую задержку закрываются обратно.

Цель игры — найти все пары карточек за минимальное количество ходов.

## 4. Игровое поле

Игровое поле состоит из 16 карточек, расположенных в сетке 4×4.

В каждой игре используется:

- 8 уникальных изображений;
- каждая картинка повторяется 2 раза;
- всего на поле находится 16 карточек.

Перед началом партии карточки должны случайно перемешиваться.

## 5. Главное меню

При запуске игры пользователь попадает в главное меню.

В главном меню должны быть:

- название игры;
- кнопка выбора набора изображений;
- кнопка выхода из игры.

После выбора набора изображений игра должна перейти на основной игровой экран.

## 6. Наборы изображений

В игре должны быть встроенные наборы карточек.

Каждый набор должен содержать минимум 8 разных изображений.

## 7. Правила игры

1. Игрок выбирает набор изображений в главном меню.
2. Игра создаёт поле 4×4.
3. Из выбранного набора берутся 8 картинок.
4. Каждая картинка дублируется, чтобы получилось 8 пар.
5. Все карточки случайно перемешиваются.
6. Все карточки отображаются закрытыми.
7. Игрок нажимает на первую карточку.
8. Первая карточка открывается.
9. Игрок нажимает на вторую карточку.
10. Вторая карточка открывается.
11. Если карточки совпали, они остаются открытыми.
12. Если карточки не совпали, они закрываются обратно через небольшую задержку.
13. После каждой попытки увеличивается счётчик ходов.
14. Игра заканчивается, когда все пары найдены.

## 8. Победа

Победа наступает, когда игрок нашёл все 8 пар карточек.

После победы на экране должно появиться сообщение:

```text
Победа!
```

Также должны быть доступны действия:

- начать игру заново;
- вернуться в главное меню;
- выйти из игры.

## 9. Счётчик ходов

В игре должен отображаться счётчик ходов.

Один ход считается после открытия двух карточек.

Пример:

```text
Ходы: 12
```

Чем меньше ходов сделал игрок, тем лучше результат.


## 10. Управление

Управление осуществляется мышью.

Основные действия:

- клик по кнопке меню — выбор действия;
- клик по карточке — открыть карточку;
- клик по кнопке перезапуска — начать игру заново;

## 11. Интерфейс игрового экрана

На игровом экране должны отображаться:

- игровое поле 4×4;
- карточки;
- счётчик ходов;
- выбранный набор изображений;
- кнопка перезапуска;
- кнопка возврата в главное меню;
- сообщение о победе после завершения игры.

Пример интерфейса:

```text

[ ? ] [ ? ] [ ? ] [ ? ]

```

## 12. Требования к карточкам

Каждая карточка должна иметь состояния:

- закрыта;
- открыта;
- найдена.

Описание состояний:

| Состояние | Описание |
| Закрыта | Игрок видит только заднюю часть карточки |
| Открыта | Игрок видит изображение карточки |
| Найдена | Карточка совпала с парой и остаётся открытой |

Игрок не должен иметь возможность повторно выбрать уже найденную карточку.

Игрок не должен иметь возможность выбрать одну и ту же карточку два раза за один ход.

## 13. Логика сравнения карточек

У каждой карточки должен быть идентификатор изображения.

Если у двух открытых карточек одинаковый идентификатор, значит пара найдена.

Пример:

```text
card_1.image_id == card_2.image_id
```

Если идентификаторы совпадают:

- обе карточки получают состояние `найдена`;
- они остаются открытыми.

Если идентификаторы не совпадают:

- карточки показываются игроку короткое время;
- затем обе карточки снова закрываются.

## 14. Задержка перед закрытием

Если игрок открыл две разные карточки, они должны закрыться не сразу, а через небольшую задержку в 1 секунду.


Во время задержки временно запретить открытие других карточек, чтобы не ломалась логика игры.

## 15. Технические требования

Проект должен быть реализован на Python с использованием библиотеки Pygame.

Минимальные требования:

- Python 3.10 или выше;
- Pygame;
- запуск игры через файл `main.py`.

Установка зависимости:

```bash
pip install pygame
```

Запуск игры:

```bash
python main.py
```

## 17. Основные модули проекта

### main.py

Главный файл запуска игры.

Отвечает за:

- инициализацию Pygame;
- создание окна;
- главный игровой цикл;
- переключение между меню и игрой.

### config.py

Файл с настройками проекта.

В нём хранятся:

- размер окна;
- цвета;
- FPS;
- размеры карточек;
- отступы;
- пути к изображениям.

### menu.py

Модуль главного меню.

Отвечает за:

- отображение кнопок;
- выбор набора изображений;
- переход к игре;
- выход из приложения.

### game.py

Модуль основной игровой логики.

Отвечает за:

- создание поля;
- перемешивание карточек;
- обработку кликов;
- проверку совпадений;
- подсчёт ходов;
- проверку победы.

### card.py

Модуль карточки.

Описывает одну карточку и её свойства:

- изображение;
- положение на экране;
- состояние;
- проверка клика по карточке;
- отрисовка карточки.

### assets.py

Модуль загрузки ресурсов.

Отвечает за:

- загрузку картинок;
- масштабирование изображений;
- подготовку наборов карточек.

## 18. Этапы разработки

### Этап 1. Подготовка проекта

- создать структуру папок;
- установить Pygame;
- создать окно игры;
- настроить игровой цикл.

### Этап 2. Главное меню

- сделать экран главного меню;
- добавить кнопки выбора наборов;
- добавить кнопку выхода;
- реализовать переход из меню в игру.

### Этап 3. Карточки

- создать класс карточки;
- загрузить изображения;
- сделать заднюю часть карточки;
- реализовать открытие карточки по клику.

### Этап 4. Игровая логика

- создать поле 4×4;
- перемешать карточки;
- реализовать выбор двух карточек;
- реализовать проверку совпадения;
- реализовать закрытие несовпавших карточек.

### Этап 5. Победа и перезапуск

- добавить проверку победы;
- вывести сообщение о победе;
- добавить кнопку перезапуска;
- добавить кнопку возврата в меню.

## 19. Критерии готовности

Проект считается готовым, если:

- игра запускается без ошибок;
- пользователь может выбрать набор изображений;
- после выбора набора открывается игровое поле;
- карточки расположены случайно;
- игрок может открывать карточки мышью;
- совпадающие карточки остаются открытыми;
- несовпадающие карточки закрываются обратно;
- счётчик ходов работает корректно;
- после нахождения всех пар появляется сообщение о победе;
- кнопка перезапуска начинает новую игру;
- кнопка возврата в меню возвращает пользователя на главный экран.
