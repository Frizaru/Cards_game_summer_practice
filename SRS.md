# Техническое задание: игра Cards на Pygame

## 1. Общая информация

**Название проекта:** Cards  
**Жанр:** Логическая игра на запоминание  
**Платформа:** Desktop  
**Язык программирования:** Python  
**Библиотека:** Pygame  

## 2. Цель проекта

Реализовать логическую игру Cards, в которой игрок должен находить пары одинаковых карточек на игровом поле.

Перед началом игры пользователь будет должен попадать в главное меню, где он сможет выбрать один из встроенных наборов изображений для карточек.

После выбора набора будет запускаться игровое поле с карточками выбранной тематики.

## 3. Основная идея игры

На игровом поле расположены закрытые карточки. Игрок открывает две карточки за один ход.

Если изображения на карточках будут совпададать, карточки будут оставаться открытыми.  
Если изображения не будут совпадать, карточки через короткую задержку будут закрываться обратно.

Цель игры — найти все пары карточек за минимальное количество ходов.

## 4. Игровое поле

Игровое поле будет состоять из 16 карточек, расположенных в сетке 4×4.

В каждой партии будет использоваться:

- 8 уникальных изображений;
- каждая картинка будет повторяться 2 раза;
- всего на поле будет находится 16 карточек.

Перед началом партии карточки будут должны случайно перемешаться.

## 5. Главное меню

При запуске игры пользователь будет попадать в главное меню.

В главном меню должны быть:

- название игры;
- кнопка выбора набора изображений;
- кнопка выхода из игры.

После выбора набора изображений игра будет должна перейти на основной игровой экран.

## 6. Наборы изображений

В игре должны быть встроенные наборы карточек.

Каждый набор должен содержать минимум 8 разных изображений.

Также должна быть отдельная картинка задней части карточки, которая отображается, пока карточка закрыта.

## 7. Правила игры

1. Игрок выбирает набор изображений в главном меню.
2. Игра создаёт поле 4×4.
3. Из выбранного набора берутся 8 картинок.
4. Каждая картинка дублируется, чтобы получилось 8 пар.
5. Все карточки случайно перемешиваются.
6. Все карточки отображаются закрытыми.
7. Игрок нажимает на первую карточку.
8. Первая карточка открывается.
9. Игрок нажимает на вторую карточку.
10. Вторая карточка открывается.
11. Если карточки совпали, они остаются открытыми.
12. Если карточки не совпали, они закрываются обратно через небольшую задержку.
13. После каждой попытки увеличивается счётчик ходов.
14. Игра заканчивается, когда все пары найдены.

## 8. Победа

Победа наступает, когда игрок найдёт все 8 пар карточек.

После победы на экране должно будет появиться сообщение:

```text
Победа!
```

Также должны быть доступны действия:

- начать игру заново;
- вернуться в главное меню;
- выйти из игры.

## 9. Счётчик ходов

В игре будет должен отображаться счётчик ходов.

Один ход будет считаться после открытия двух карточек.

Чем меньше ходов сделает игрок, тем лучше будет результат.

## 10. Управление

Управление будет осуществляется мышью.

Основные действия:

- клик по кнопке меню — выбор действия;
- клик по карточке — открыть карточку;
- клик по кнопке перезапуска — начать игру заново;

## 11. Интерфейс игрового экрана

На игровом экране должны отображаться:

- игровое поле 4×4;
- карточки;
- счётчик ходов;
- выбранный набор изображений;
- кнопка перезапуска;
- кнопка возврата в главное меню;
- сообщение о победе после завершения игры.

## 12. Критерии готовности

Проект будет считаться готовым, если:

- игра запускается без ошибок;
- пользователь может выбрать набор изображений;
- после выбора набора открывается игровое поле;
- карточки расположены случайно;
- игрок может открывать карточки мышью;
- совпадающие карточки остаются открытыми;
- несовпадающие карточки закрываются обратно;
- счётчик ходов работает корректно;
- после нахождения всех пар появляется сообщение о победе;
- кнопка перезапуска начинает новую игру;
- кнопка возврата в меню возвращает пользователя на главный экран.
